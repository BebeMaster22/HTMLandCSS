Style attribute

The style attribute is used to define style information within a document and how a browser should render a document. You can apply a style to the whole document by inserting it in the head, however, to apply it to specific elements, you must utilise the class attribute which will be discussed further in the following paragraph. Simple applications of the style attribute include colour in the following syntax: style="color:blue" or < style> h1{color:red} As for border simply swap out colour for border and specify the parameters. For example, style="border: 1px black", this will make the border 1 pixel and black in colour.

Class attribute

The class attribute is used to specify a class within the style sheet. For example, I could create a style element p.solid(border-style:solid) and could later apply it in specific elements by referring to it as such: < p class="solid"> to create a class, start within a style element and begin by naming the style class after a full stop. This is the name that you will refer to the class as. For example I might name my font colour class as '.blue'. Following this, you must specify the parameters of the class as right now, all the browser knows is the name of the class. To do this, you must enclose the parameters in {} and then specify the type of parameter for example, {colour:blue;}. The style must be finished by a semicolon, afterwhich you can add more styles. For example, {border-style:dotted;colour:blue;} as such, the style in this case, 'blue' will result in the text or whatever being surrounded by a black dotted border and blue in colour. However, in this case, the border and text will be blue, as such, it is better to be more specific and seperate styles. So in this case, I should have made a seperate style for the font colour and applied it seperately. This works since elements can have multiple classes assigned to it and different elements can share classes. To do this, simply seperate two style classes by space. Some types of style class include: border-style, color, backround-color, font-size, and padding. Here is a full example of the correct syntax. What do dashes, single dots, and double dots mean in paths?

The dashes, single dots, and double dots seen in paths are syntaxes for paths which help organise different paths, directories and whatnot into neat categories. First and foremost, the slash which can come in the form of a forward or backward slash indicates seperations between folders or file names. For example, bebe_files/ directs you to a directory with that name while bebe/files directs you to a directory called files within a directory called bebe. Second of all, a single dot directs you to the current directory while two dots directs you to the parent directory.

Link States

Links change states. For example, when you click on a link, it changes colour as it has changed states. There are five distinct states a link could be in. First is the default state, ':link' which displays a link which has not been clicked or visited yet. Second is the ':visited' state, which occurs when the user has clicked or visited a link. By default, the link will turn purple when the state becomes ':visited'. Third is the ':hover' state, which occurs when a user is hovering their cursor over the link. Fourth is ':focus', which occurs when a user focuses on a link. And lastly is ':active', which occurs when the link is being activated by a user. When using these states to style links, you must follow a particular order when writing them in CSS. The order is as follows.

  1. link
  2. visited
  3. hover
  4. focus
  5. active

Blockquotes, Inline Quotes, and Citation

The 'blockquote' element and 'q' (Inline quote) element both allow you to quote something. It visually differs from surrounding text by providing an indentation to the quote. Meanwhile, the 'cite' element allows you to cite the quote either by URL or simply title. The difference however, between blockquoting and Inline quoting is that they are simply the same visually, the only difference is the information it provides behind the scenes. Inline quotes are made for smaller quotations a few sentences more or less, while blockquotes can be up to a paragraph long. As such, the purpose of these two elements is for the sake of semantics as it helps the browser understand what you are trying to convey within the html as it doesn't gather information explicitly by reading the content of your elements, rather, the element themselves to get a sense of what you are trying to convey. The syntax for these elements is simple. For blockquotes and inline quotes simply nest the quote in your desired element, < blockquote> or < q>. However, for citation, you should put the element outside of the quote if you wish it to be visible. This is better for referencing titles. However, you can also insert the cite element within the opening brackets of a blockquote or inline quote element which will allow you to cite the URL behind the scenes.

Basic Structure of CSS

The basic syntax of all CSS is made up of a selector and a declaration block. selector {property:value;} The selector is used to target various html elements that can be subjected to modification and styling which occurs in the declaration block {}. The property determines which feature is being styled e.g backround-color. The value is then the modifcation applied to the property. In this case it could be purple or white. After every property, you must have a colon, and after every value, you require a semicolon.

Selectors

There are different types of selectors. The type selector applies the value and property to a type of element. If you added a border to the table element, every table element would have a border. The next type is the ID selector where you apply the property and value to a certain ID. Every ID must start with a hashtag. Followingly, is the class selector which performs similarly to an ID, except instead of a preceding hashtag, you require a full stop. An ID and class can be applied as an attribute to any opening bracket.

Meta Viewport Element

The meta viewport element provides the browser with instructions on how to control the pages dimensions and scaling on different devices and screen sizes. The meta viewport is inserted in the head element. The syntax is as follows: < meta name="viewport" content="width=device-width, initial-scale1.0"> The wdith=device-width tells the browser to match the width of the page to the width of the screen. The initial-scale1.0 tells the browser to set the inital zoom level at 100% when the page is initially loaded.

Width and Height Elements

The width and height elements define the width and height of an element using measures of px, %, or viewport units(vw,vh). In order to insert a simple box with heigh and width elements, it is best to create a class and the class as an attribute to a div or section.

Combinators

Connectors are the characters which combine different elements to style as one. The first type of combinator is the descendent combinator. The descendent combinator is a space between two element selectors. It's function is to apply the style to the second element in the selector IF it is nested within the first element. Take this example: figure p {border:"";} in this case, the style will only apply to the p element under the condition that it is the child of a figure element. The descendent combinator requires a space between the first and second element. The next time of combinator is the child combinator, >. The function of the child combinator applies a style to the direct child of an element. The parent element precedes the greater than symbol while the direct child is after said symbol. Followingly, we have the 'next-sibling' combinator, +, which targets the next direct sibling of an element for stylising. Say we have a h1, h2, and h3 element in the same div element. If I wanted to target the h3 element for stylisation using the sibling combinator, I'd have to precede the + sign with the h2 element as the h3 element is the direct sibling of the h2 element. As a result, the style would only apply to the h3 element. Finally, we have the 'subsequent-sibling' combinator, ~, which works similarly to the next-sibling combinator. Except, the subsequent sibling combinator targets all subsequent siblings of a chosen element of a chosen element. Say I had a h2 element followed by three h3 elements and two h2 elements. If I wanted to target all p elements following the h2 element, all I would have to do is precede the ~ with the h2 element, and after the ~ a p element in the following syntax: h2~p{"":"";}. Note that this will only occur if they are siblings nested within the same elements. Additionally, note that putting this style in the header would apply to all instances which match the requisites of the style. As such, it is better to create a class or ID for this style which can be applied individually. This can work by applying an ID or class to one of the elements involved in the combinator. For example, in the opening brackets of my chosen h2 element, I could make the class="target". Then, within my style element I could add this class in the following syntax: h2.target~p{"":"";}. As a result, the style will only apply to the h2 element under the class of '.target'.

Inline Block Elements

There are three primary display behaviours when it comes to elements. Inline, block, and inline-block. Block elements take up the full width it can. They start on a new line and are adjustable. For inline elements, they flow around the surrounding elements and only take up the necessary amount of space, they are not adjustable however. On the other hand, Inline-Block-Elements are inline elements, just with the ability to adjust their parameters. The syntax is normal when it comes to styling, however, you must specify the display in the value and property block as such: { display:inline-block}. Additionally, it is recommended to use the span element which is essentially a container of the same respect as div, just that it is geared towards styling.

Margin and Padding

The margin element allows you to add a margin to a certain side or all sides of an element. Of course you can add it inline as an attribute, however it is best to simply add it to every element as a style. The different variations of margin is as follows: Margin, margin-left,margin-right,margin-top, and margin-bottom. The margin element on its own adds a margin of a specified size to all sides of the element. While the other margin elements are added to certain sides. Take this syntax: { margin: 10px 40px} the margin would add 10px to the top and bottom and 40px to the left and right. If we were to add a third value, the 10px would be added to the top, 40px to the left and right, but the new value would be added to the bottom. Adding a fourth value would change the individual sizes for each margin side in the order: top,right,bottom,left. As for padding, the element variations are the same as the margin element.

Gradients

Gradients are used in CSS to make smooth transitions between different colours. There are two types of CSS gradients: Linear, and radial gradient. Linear gradients transition colours across a straight line. The direction and colours are defined within the syntax. The syntax is as follows: selector {background:linear-gradient(to right, blue, green);} In this syntax, the linear gradient value is used. Within its brackets contains the parameters for the gradient. The direction is specified first, followed by the colours. In this case, yellow would blend into red from left to right. On the other hand, radial gradients are gradients that stem from one point. The syntax is as follows: selector {background:radial-gradient(circle closest-side at center, yellow, skyblue);} In this case, the first term within the brackets, circle, determines the shape of the gradient, then, the second term 'closest side' determines the gradient's ending size, in this case the closest side. Followingly, the third term 'at center' determines where the gradient will begin, in this case, at the center. And finally, you have the gradient colours.
